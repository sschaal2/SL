304 \remarks
305 
306  Reads a frame from the serial port.
307  Note: all date comes as ASCII characters!!! Weird .....
308 
309  The following characters demark the frame:
310  start of frame : ^
311  end of frame : \n
312  start of camera 2: /
313  end of channel : ;
314  end of number : space
315 
316  For the following readings, the DBvision hardware needs to
317  be set to DrScheme communication. This means:
318  - report frame counter (otherwise this function reports errors)
319  - only report x and y of the blobs, no other data
320  (this could be changed if needed)
321  - only allow one object per channel
322  (can be improved in the future, too, but requires to use
323  temporal reasoning to avoid confusing two objects with
324  the same color, and also occlusions)
325 
326  Currently, blob numbering coincides with the the color channel
327  number.
328 
329  Thus, an empty frame would look like:
330  1213;;;;/;;;;/
331 
332  A valid frame with one blob would looke like:
333  4475460 356 269;;;;/238 174;;;;/
334 
335  *******************************************************************************
336  Function Parameters: [in]=input,[out]=output
337 
338  \param[in] buffer : charcter buffer with frame
339  \param[in] n_buffer : number of characters in frame
340 
341  returns the number of bytes read

343  ******************************************************************************/
