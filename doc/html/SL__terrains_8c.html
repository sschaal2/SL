Detailed Description

=============================================================================

Author
Date
Remarks
A set of functions to read, store, and process information about
terrains for walking, originally developed in the context of the
LittleDog project, and then generalized to a generic feature in SL.
The idea is to store complex terrains as a grid file in x,y,z, which
can be queried very fast. Visualization can be done in a rudementary
way without too much CPU costs

computeTerrainNormal
Date
April 2006
Remarks
    Computes the terrain normal at a certain location in a terrain
    using a certain neigborhod of points

Function Parameters: [in]=input,[out]=output

Parameters
[in]ID: terrain array index ( between 1 and MAX_TERRAIN)
[in]ix: the integer index of the x coordinate of terrain location
[in]iy: the integer index of the y coordinate of terrain location
[in]nx: the number of x neighbors to consider for regression
[in]ny: the number of y neighbors to consider for regression
[in]down: 1=no down sampling, 2=every other point in x and y direction is chosen, etc.
[in]cflag: compute for contact representation
[in]fflag: compute for foothold representation
[out]tinfo: the terrain info structure
